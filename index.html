What is the Prometheus Design System?

The Prometheus Design System (@prometheus-io/design-system) is an extraction and normalization of the UI patterns, tokens, and components found in the Prometheus monitoring tool's web interface.

It solves concrete problems found during a deep codebase audit:

  • Fragmented health badge system → unified StatusBadge component
  • Dual border inconsistency (AlertsPage vs RulesPage) → HealthPanel
  • 4 different filter bars → FilterToolbar
  • Hardcoded light-mode-only colors → light-dark() CSS throughout
  • Missing dark mode on SeriesName hover → fixed
  • Hardcoded spacing values → named tokens

Design Philosophy

"Derive, not invent"

Every token, component, and pattern in this design system was derived from the actual Prometheus codebase — not invented from scratch. The system documents what already exists and resolves inconsistencies where the same pattern was implemented differently across pages.

This means:

  • System UI fonts, not custom web fonts (this is a developer tool)
  • DejaVu Sans Mono for code (the Prometheus-specific choice)
  • Mantine's default spacing and radius scales (intentionally not customized)
  • The header is always dark — this is Prometheus brand identity, not a bug

About This Project

This design system is an independent proposal — it is not currently an official Prometheus project or a Grafana Labs initiative. It was created by extracting and normalizing patterns from the prometheus/prometheus web UI codebase.

The goal is to propose this work to the Prometheus maintainer community for adoption. The path forward:

  1. Gather feedback from Prometheus maintainers on this documentation and component library
  2. Open upstream PRs to prometheus/prometheus for the inconsistency fixes documented here
  3. Propose the design system as an official sub-project through the CNCF governance process
  4. Transfer ownership to the prometheus GitHub organization if accepted

Decision-making: All design decisions are documented in Architecture Decisions. Proposed changes follow the process described in Contributing.

Installation

# Install the design system
npm install @prometheus-io/design-system

# Peer dependencies (install alongside)
npm install @mantine/core @mantine/hooks @tabler/icons-react react react-dom

Quick Start

// In your app entry point (e.g., main.tsx):
import "@mantine/core/styles.css";
import "@prometheus-io/design-system/styles.css";

import { PrometheusThemeProvider } from "@prometheus-io/design-system";

function App() {
  return (
    <PrometheusThemeProvider defaultColorScheme="auto">
      <YourApp />
    </PrometheusThemeProvider>
  );
}

Design Tokens

All tokens are exported as TypeScript constants and CSS custom properties (--prom-*).

import {
  BRAND_ORANGE,     // "#e6522c"
  HEADER_BG,        // "rgb(65, 73, 81)"
  healthColors,     // { ok: { badgeBgLight, ... }, err: {...}, ... }
  fontFamilies,     // { body, mono, completionTooltip }
  spacing,          // { xs, sm, md, lg, xl } → CSS var references
  motion,           // { hoverReveal, accordionChevron, ... }
} from "@prometheus-io/design-system";

Prometheus 3.x Compatibility

This design system is derived from the current Prometheus web UI codebase. Active work is planned to track Prometheus 3.x features:

3.x FeatureDesign System StatusTarget
Native histogram displayPlanned — new HistogramChart componentv0.2.0

The design system aims to be an active partner in the Prometheus 3.x UI work, not a retrospective cleanup exercise. Component proposals for new 3.x features will be tracked in the GitHub issue tracker.
