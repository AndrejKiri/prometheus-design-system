EndpointLink

Scrape endpoint URL display with query parameter badges.

Description

Scrape endpoint URL display with query parameter badges. Parses the URL and renders query parameters as small gray pill badges alongside the clickable link.

Where this component exists in the Prometheus UI
View in Prometheus UI → /targets

EndpointLink on the Targets page — clickable scrape endpoint URLs

Visual Examples

EndpointLink Example
http://localhost:9090/metrics
http://localhost:9090/api/v1/query query=up time=1234567

Code Example

import { EndpointLink } from "@prometheus-io/design-system";

<EndpointLink endpoint="http://localhost:9090/metrics" />
<EndpointLink endpoint="http://localhost:9090/api/v1/query?query=up" />

Accessibility

  • Link is a standard <a> element
  • Query parameter badges are purely visual — not interactive

Do / Don’t

✓ Do
  • Displaying scrape target endpoints
  • Any URL display that benefits from parsed query params
  • Use EndpointLink for all scrape target endpoint display.
✗ Don’t
  • Generic link display without query parameter parsing
  • Navigation links (use NavButton)
  • Don't manually parse URLs and render query params as badges.
