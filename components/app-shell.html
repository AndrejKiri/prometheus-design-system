PrometheusAppShell

Application layout shell with the intentionally-always-dark header.

Description

Application layout shell with the intentionally-always-dark header. Wraps Mantine AppShell with Prometheus-specific header styling, navigation, and responsive behavior. The header uses rgb(65, 73, 81) background and never adapts to theme.

Code Example

import { IconDatabase } from "@tabler/icons-react";

<PrometheusAppShell
  logo={<YourLogo />}
  navLinks={
    <>
      <NavButton href="/query" icon={<IconDatabase size={16} />}>Query</NavButton>
      <NavButton href="/alerts">Alerts</NavButton>
    </>
  }
  actions={<><ThemeToggle /><SettingsPanel groups={[...]} /></>}
>
  <Routes>...</Routes>
</PrometheusAppShell>

Accessibility

  • Uses Mantine AppShell for proper landmark structure
  • Responsive: drawer navigation on mobile, inline on desktop
  • Burger button appears on mobile with proper aria-label
  • Navigation uses semantic <nav> element

Do / Don’t

✓ Do
  • Top-level application wrapper
  • Main layout structure with header + nav + content area
  • Use PrometheusAppShell as the top-level layout. The always-dark header is intentional brand identity.
✗ Don’t
  • Embedded widgets (they don't need the full app shell)
  • Pages without navigation
  • Don't try to make the header respond to light/dark mode. Don't hardcode header colors elsewhere.
