Changelog

Version history and release notes for the Prometheus Design System.

Versioning
This project follows Semantic Versioning. The patch number is incremented automatically on each push. Major versions reflect significant shifts in scope or approach.

v0.2 — Documentation maturity

April 2026

A focused pass on making the docs site honest, navigable, and useful. Speculative and placeholder content was replaced with real source references. Navigation was restructured from scratch — collapsible sidebar, sticky TOC, icons throughout. All 19 component pages received Do/Don't usage guidelines. The site now clearly signals its WIP state and links every action item back to a GitHub issue.

v0.1 — Foundation

March 2026 · tagged v0.1.0

First complete draft of the Prometheus Design System. The work began as a deep audit of the prometheus/prometheus Mantine-based UI, identifying recurring patterns and inconsistencies. From that audit, 19 reusable components were extracted and documented, a --prom-* design token layer was defined, 9 audited inconsistencies were resolved, and a full static documentation site was shipped alongside a Figma bootstrap plugin that sets up the component library in one run.

v0.1.2

  • Figma plugin rebuilt to match design system specs exactly — component scaffolds now reflect real Prometheus component structure
  • Fixed runtime crashes in component creation logic
  • Fixed Inter font style lookup (SemiBold → Semi Bold)
  • Fixed auto-layout sizing: layoutSizingVertical must be set after appending to parent
  • Fixed FILL sizing: use layoutSizingVertical not counterAxisSizingMode
  • Fixed Figma plugin component layout and sizing issues

v0.1.1

  • Figma bootstrap plugin shipped — automates design token variables, color styles, text styles, and component scaffolds
  • Dev Mode support added to the plugin
  • Proper attribution added to the project
  • Incorrect AI-generated links fixed in documentation

v0.1.0

  • Extracted 19 components from the prometheus/prometheus Mantine UI codebase: StatusBadge, HealthPanel, LabelBadge, FilterToolbar, InfoCard, InfoPageLayout, DataTable, KeyValueTable, CodeBlock, SeriesName, EndpointLink, EmptyState, ErrorAlert, PoolAccordion, NavButton, StateMultiSelect, SettingsPanel, ThemeToggle, PrometheusAppShell
  • Resolved 9 audited inconsistencies — see Inconsistency Log
  • Unified 3 different health badge implementations → StatusBadge
  • Consolidated 4 duplicate filter bars → FilterToolbar
  • Fixed localStorage key collision between AlertsPage and RulesPage
  • Added dark mode support to SeriesName hover colors
  • Normalized panel border color steps across AlertsPage and RulesPage
  • Fixed navbar text visibility gap between sm and md breakpoints
  • Defined --prom-* CSS custom properties for colors, spacing, typography, and motion
  • Health color system: 5-state semantic palette (ok/err/warn/info/unknown) with light and dark mode values
  • Complete documentation site: component pages, design tokens, icon inventory, UI patterns, Figma guide, migration, inconsistency log
  • Accessibility documentation (WCAG AA compliance target)
  • Pattern documentation for 28 identified UI patterns
