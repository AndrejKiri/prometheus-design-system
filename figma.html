Figma Library Setup Guide

Follow these steps to create a Prometheus design library in Figma that mirrors the codebase. Estimated time: 30-45 minutes for initial setup.

Prerequisites
Figma Professional plan or higher (to publish team libraries). Familiarity with Figma variables and component variants.

Step 1: Create the Library File

Create a new Figma file named Prometheus Design System. Organize with these pages:

  • Cover — title card with version number
  • Variables — workspace for variable collections
  • Text Styles — text style definitions
  • Components — all component masters
  • Icons — Tabler icon subset

Step 2: Create Variable Collections

Open the Variables panel (⌘/Ctrl + Shift + V) and create these collections:

  1. Brand — brand/orange (#e6522c), brand/header-bg (rgb(65,73,81)), brand/primary (#228be6)
  2. Health Status — add two modes (Light / Dark). Create variables for each status (ok, err, warn, info, unknown) with -bg, -text, -border suffixes. Reference the tier-1 primitive values from the Color Variable Structure section below.
  3. Surface — surface/card-bg, surface/codebox-bg, surface/accordion-bg (with Light/Dark modes)
  4. Spacing — spacing/xs (8), spacing/sm (12), spacing/md (16), spacing/lg (20), spacing/xl (32)
  5. Radius — radius/default (4), radius/sm (2), radius/md (8), radius/lg (16), radius/pill (9999)

Step 3: Create Text Styles

Create all text styles using the naming convention from the Typography section above. Set the font family to your system's default sans-serif for body styles and DejaVu Sans Mono for code styles.

Step 4: Build Primitive Components

Start with the smallest building blocks:

  1. StatusBadge — create a frame (auto-layout, horizontal, center-aligned). Add text layer using label/badge style. Set fill to health/ok-bg variable. Add variants for all status values and sizes.
  2. LabelBadge — similar pill shape, using code/label text style. Two variants: default and discovered.

Step 5: Build Panel Components

  1. HealthPanel — auto-layout frame with 5px left border. Add nested StatusBadge. Create variants for each health status and collapsed/expanded states.
  2. ErrorAlert — auto-layout with icon + text. Variants for severity (error, warning, info) and has-title (true, false).
  3. EmptyState — centered vertical layout with icon, message, description, and optional action.

Step 6: Build Data Display Components

  1. DataTable — use auto-layout rows. Create header row and data row sub-components. Variant for empty state.
  2. KeyValueTable — two-column layout with bold keys and regular values.
  3. CodeBlock — frame with monospace text, gray background. Variants for overflow mode.
  4. SeriesName — use monospace text styles with syntax coloring. Variants: plain and formatted.

Step 7: Build Navigation Components

  1. NavButton — auto-layout with optional icon and label. Variants for active state.
  2. ThemeToggle — 32px square icon button. Two variants: light mode (moon icon) and dark mode (sun icon).

Step 8: Build Composite Components

  1. InfoCard — card frame using surface/card-bg, border, shadow/xs. Nest icon + title header and content slot.
  2. FilterToolbar — horizontal auto-layout containing search input, state multi-select, and icon buttons. Use boolean variants for show/hide controls.
  3. PoolAccordion — nest HealthPanel items. Include stats badges in the header.
  4. SettingsPanel — card with grouped checkbox/number settings.

Step 9: Build the App Shell

Create the PrometheusAppShell as a top-level frame:

  1. Header bar (56px height, rgb(65,73,81) background) with logo, NavButtons, and action icons
  2. Content area slot below the header
  3. Variants for navbar state (inline, drawer-open, drawer-closed) and theme (light, dark)

Step 10: Apply Variable Modes

For every component that changes between themes:

  1. Select the component set
  2. In the right panel, under "Variables", bind fill/stroke/text colors to the semantic variables
  3. Figma will auto-swap values when you switch between Light and Dark modes on the parent frame

Step 11: Publish the Library

  1. Go to Assets panel → click the book icon → Publish
  2. Add a meaningful description and version note
  3. Team members can now enable this library in their files via Assets → Team Library

Step 12: Maintain Parity

After publishing, keep the library in sync with code changes:

  • When a new prop is added to a component, add a corresponding Figma variant property
  • When token values change, update the Figma variable values
  • Use the Figma bootstrap plugin to automate initial variable and color style creation
  • Version the library using Figma's built-in branching or a changelog on the Cover page

Color Variable Structure

Follow a three-tier color variable architecture:

Tier 1: Primitive Colors

Raw Mantine color scale values. These are the atomic building blocks.

primitive/green-1 → #ebfbee
primitive/green-3 → #8ce99a
primitive/green-8 → #2f9e44
primitive/green-9 → #2b8a3e
primitive/red-1   → #fff5f5
...

Tier 2: Semantic Colors

Named by meaning, referencing primitives. Mode-aware (light/dark values).

semantic/health-ok-bg       → primitive/green-1 (light) | primitive/green-9 (dark)
semantic/health-ok-text     → primitive/green-9 (light) | primitive/green-1 (dark)
semantic/health-ok-border   → primitive/green-3 (light) | primitive/green-8 (dark)
semantic/health-err-bg      → primitive/red-1 (light)   | darken(primitive/red-9) (dark)
...

Tier 3: Component Colors

Component-specific aliases referencing semantic colors.

component/status-badge-ok-bg    → semantic/health-ok-bg
component/health-panel-ok-border → semantic/health-ok-border
component/label-badge-bg         → primitive/gray-1 (light) | primitive/gray-8 (dark)
component/codebox-bg             → primitive/gray-1 (light) | primitive/gray-9 (dark)
...

This three-tier system ensures that changing a semantic color automatically propagates to all components that reference it.

Bootstrap Plugin

The Prometheus DS Bootstrap Plugin automates the initial Figma library setup. It creates all variable collections, text styles, effect styles, and component scaffolds in one click.

Plugin Files
The plugin source is in figma-plugin/manifest.json and figma-plugin/code.js. Load it in Figma via Plugins → Development → Import plugin from manifest.

What the Plugin Creates

CategoryItems Created
Variable CollectionsBrand (3 colors), Health Status (5 statuses × 3 roles × 2 modes), Surface (3 surfaces × 2 modes), Spacing (5 values), Radius (5 values)
Text Styles13 text styles across heading, body, label, code, and stats categories
Effect Styles6 elevation styles (none through xl)
Component Scaffolds19 named component frames with auto-layout, matching the React component library

Installation

  1. Clone or download the figma-plugin/ directory
  2. In Figma, go to Plugins → Development → Import plugin from manifest
  3. Select the manifest.json file
  4. Run the plugin from Plugins → Development → Prometheus DS Bootstrap

Plugin API Methods Used

MethodPurpose
figma.variables.createVariableCollection()Create variable collections (Brand, Health, Surface, Spacing, Radius)
figma.variables.createVariable()Create individual variables within collections
collection.addMode()Add Light/Dark modes to themed collections
variable.setValueForMode()Set per-mode values for themed variables
figma.createTextStyle()Create named text styles
figma.createEffectStyle()Create named effect (shadow) styles
figma.createComponent()Create component scaffolds
figma.createPage()Create organized pages
figma.loadFontAsync()Load fonts before setting text
figma.notify()Show progress notifications
figma.closePlugin()Clean shutdown after completion

Post-Plugin Steps

After running the plugin, manually refine:

  1. Replace Inter font with your system font stack if preferred (the plugin uses Inter as a Figma-compatible stand-in for the system UI font)
  2. Add variants to each component scaffold following the variant tables above
  3. Bind component fills/strokes to the created variables
  4. Publish the library for team access
