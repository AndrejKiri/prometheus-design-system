YAML Code Block

Syntax-highlighted configuration display

✓ DoDeclare an explicit monospace font stack (e.g. 'ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, monospace') to ensure consistent rendering across OS (fixes INC-009).
✗ Don'tDon't use 'monospace' as the sole font-family value — it resolves to different fonts on macOS, Windows, and Linux.
✓ DoKeep hljs-attr and hljs-string in the same hue families across light and dark to avoid color-role swaps (fixes INC-008).
✗ Don'tDon't let highlight.js token colors swap semantic roles (keys ↔ strings) between themes.

Description

Full config rendered as a <pre><code class='language-yaml'> block with highlight.js syntax highlighting inside a Mantine Card. Font is browser-default 'monospace' with no explicit stack (INC-009). Syntax token colors swap roles between light and dark themes (INC-008).
