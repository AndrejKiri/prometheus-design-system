Health Status Badge

Target health status indicator

✓ DoAuthor the DOM text in uppercase ('UP', 'DOWN') to match visual output — removes reliance on textTransform (INC-007).
✗ Don'tDon't use textTransform to uppercase lowercase source text — it breaks copy-paste and screen-reader output (INC-007).
✓ DoUse the ok/error status tokens for background and text — never hard-code the hex values.
✗ Don'tDon't add a third state (e.g. 'UNKNOWN') without also adding the corresponding token pair.

Description

Green/red pill badge shown on each scrape target to indicate health status ('UP' or 'DOWN'). Fully rounded (pill radius). Light/dark swap the fg and bg values. Note: DOM contains lowercase 'up'/'down' but CSS uppercases via textTransform — see INC-007.

Accessibility

ARIAConsider aria-label='Health: UP' so status is explicit to screen readers.
ContrastLight: rgb(43,138,62) on rgb(211,249,216) ≈ 4.1:1 — marginally passes AA for large text. Dark: rgb(211,249,216) on rgb(43,138,62) — similar.
Screen readerCurrent lowercase DOM text 'up'/'down' may be announced differently than the visible 'UP'/'DOWN' — see INC-007.
