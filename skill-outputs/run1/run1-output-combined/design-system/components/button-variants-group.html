Button Variants Group

Multi-variant query toolbar

✓ DoGroup related controls visually using gaps and Mantine Group component.
✗ Don'tDon't add more primary-filled buttons to this toolbar — the Execute button is the only call-to-action.
✓ DoReserve the filled/primary variant for the single most important action.
✗ Don'tDon't replicate this dense toolbar pattern elsewhere — it is specific to the query surface.

Description

The dense toolbar on the PromQL query page containing nine visually distinct button variants: Execute (primary filled), metrics explorer (subtle), tree-view toggle, result tabs (segmented control), timestamp toggle, history dropdown, and add-panel button.

complex input /query

Design variants

subtle

Subtle text-only button used for secondary actions (metrics explorer, tree-view toggle).

Tokens: brand-blue-light, label/button

segmented-control

Tab-style segmented control for result view (Graph / Table / Explain).

Tokens: brand-blue-light, label/button, sm

action-icon

Icon-only ActionIcon for compact secondary actions (history, add panel).

Tokens: brand-blue-light
