Mantine Alert Info Callout

Empty-state info alert

✓ DoUse this pattern consistently for all empty states across the app.
✗ Don'tDon't use this for error states — use the error badge/status pattern instead.
✓ DoInclude a meaningful message explaining why the list is empty (e.g. 'No rules found' not just 'Empty').
✗ Don'tDon't add a border to the callout — it does not have one in the current implementation.

Description

Shared empty-state pattern rendered as a Mantine Alert in 'info' color. Appears on /alerts, /rules, and /alertmanager-discovery when there is no data to display. Provides a subtle blue tint in both themes.

Accessibility

ARIAMantine Alert renders with role='alert' by default.
ContrastLight: rgb(24,100,171) on rgba(34,139,230,0.1) ≈ 4.7:1 — passes AA. Dark: rgb(116,192,252) on rgba(34,139,230,0.15) — verify contrast.
Screen readerrole='alert' causes screen readers to announce the content. Use only for genuinely important empty states.
